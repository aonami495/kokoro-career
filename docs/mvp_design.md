## プロジェクト概要

### ミッション

**「配慮の可視化」で、精神障害者と企業の"安心できるマッチング"を実現する**

### 背景

- 2026年7月に法定雇用率が2.7%に引き上げ（従業員37.5人以上の企業が対象）[1]
- 精神障害者の就職件数が前年比11.1%増と急伸[2]
- 既存の大手サービス（LITALICO、atGP等）は首都圏中心で、地方の中小企業との接点が弱い[3]
- **最大の課題**：企業が「どんな配慮をすればいいか分からない」→ミスマッチ→離職

### ソリューション

企業が提供できる「配慮」をタグ化し、求職者が自分に必要な配慮で企業を検索できるマッチングプラットフォーム。

## 市場機会とターゲット

### 市場規模

- 障害者雇用支援サービス市場：**7,500億円（2022年）**[4]
- 精神障害者の就職件数：**115,609件/年（過去最高）**[2]
- 法定雇用率未達成企業：**約52%**（2024年時点）[5]

### ターゲットユーザー

#### ペルソナ1：求職者「佐藤美咲さん（28歳）」

- **障害種別**：うつ病（精神障害者保健福祉手帳2級）
- **状況**：
  - 前職を体調不良で退職後、1年間の療養を経て就職活動中
  - 就労移行支援事業所に通所中
- **課題**：
  - 企業の「障害者雇用枠」に応募しても、実際の配慮内容が分からず不安
  - 面接で「どんな配慮が必要ですか？」と聞かれても答えづらい
  - いきなり正社員は不安、まず短期間試したい
- **ニーズ**：
  - 「静かな環境」「指示は文書で」など具体的な配慮を提供する企業を探したい
  - 1～2週間の職場体験実習で相性を確かめたい

#### ペルソナ2：採用担当者「田中健太さん（35歳）」

- **企業規模**：従業員45名のIT企業（東京）
- **状況**：
  - 2026年7月の法定雇用率引き上げで、初めて障害者雇用が義務化
  - 障害者雇用の経験なし、何から準備すべきか分からない
- **課題**：
  - ハローワークや大手求人サイトは敷居が高い
  - 「合理的配慮」の具体例が分からない
  - 採用してもすぐ辞められたら困る
- **ニーズ**：
  - 自社で提供できる配慮を選ぶだけで求人を出せるツール
  - まずは短期の体験実習でお互いの相性を確認したい
  - 採用後の定着支援のサポートが欲しい

## コアコンセプト：3つの差別化ポイント

### 1. 配慮タグによる「可視化マッチング」

**課題**：企業も求職者も、抽象的な「合理的配慮」の内容が分からない

**解決策**：配慮を20～30種類のタグに分類し、選択式でマッチング

**配慮タグの例**

| カテゴリ               | タグ例                                                                      |
| ---------------------- | --------------------------------------------------------------------------- |
| **作業環境**           | 静かな個室あり / パーティション席 / 在宅勤務可 / フレックスタイム           |
| **コミュニケーション** | 指示は文書で / 1on1定例あり / チャットツール使用 / 報連相のフォーマットあり |
| **勤務時間**           | 時短勤務可（週20h～） / 通院配慮あり / 休憩自由                             |
| **サポート体制**       | ジョブコーチ常駐 / メンター制度 / 産業医面談あり                            |

**実装イメージ**
求職者：「静かな環境」「指示は文書で」「時短勤務可」を選択
　　↓ マッチングアルゴリズム
企業：上記3つをすべて満たす企業を優先表示（マッチ度90%）

---

### 2. 「体験実習マッチング」機能

**課題**：いきなり正式雇用だとミスマッチが多く、離職率が高い

**解決策**：1～2週間の「職場体験実習」を前提としたマッチング

**フロー**

1. 企業が「実習受け入れ可」の求人を掲載
2. 求職者が応募（就労支援事業所の支援員も関与可）
3. 実習期間中、企業・求職者・支援員が日報を共有
4. 実習後、3者で振り返りミーティング
5. 双方合意なら本採用へ

**メリット**

- 企業：採用リスク軽減、実際の業務適性を確認できる
- 求職者：安心して職場を体験できる、無理なら辞退も可能

---

### 3. 全国対応×中小企業特化

**課題**：大手プラットフォームは首都圏・大企業中心で、地方の中小企業とのマッチングが弱い

**解決策**：

- 全国対応（リモート可能な求人も含む）
- **中小企業向けの簡単セットアップ**（30分で求人掲載完了）
- 自治体の助成金情報を自動表示
- 「障害者雇用はじめてガイド」を無料提供

## MVP機能設計（Phase 1: 3ヶ月）

### Phase 1の目標

**「配慮タグマッチング」と「体験実習」の2つのコア機能を検証し、10組のマッチング成立を目指す**

---

### 機能一覧

#### 1. ユーザー登録・認証（Week 1-2）

**対象ユーザー**

- 求職者（精神障害者）
- 企業（採用担当者）
- 就労支援事業所（支援員）※Phase 2で本格実装

**実装内容**

- Devise gemによるメール認証
- ユーザー種別の選択（求職者/企業/支援員）
- プロフィール登録
  - 求職者：障害種別、障害者手帳の有無、希望勤務地、希望職種
  - 企業：会社名、所在地、従業員数、業種

**工数**：3日

---

#### 2. 配慮タグ管理システム（Week 2-3）

**マスタデータ作成**

- あらかじめ用意した配慮タグ（20～30種類）
- カテゴリ：作業環境 / コミュニケーション / 勤務時間 / サポート体制

**求職者側の機能**

- 自分に必要な配慮タグを複数選択
- 「最重要」「あると嬉しい」の2段階で優先度付け

**企業側の機能**

- 自社で提供可能な配慮タグを選択
- 各タグの詳細説明を記入（任意）

**実装詳細**

# モデル設計

class AccommodationTag < ApplicationRecord

# 例: "静かな環境", "指示は文書で", "時短勤務可"

end

class JobSeekerAccommodation < ApplicationRecord
belongs_to :job_seeker
belongs_to :accommodation_tag
enum priority: { required: 0, preferred: 1 }
end

class CompanyAccommodation < ApplicationRecord
belongs_to :company
belongs_to :accommodation_tag

# 提供可能な配慮の詳細説明

end

**工数**：5日

---

#### 3. 求人登録機能（Week 3-4）

**企業側の求人登録**

- 求人タイトル、職種、業務内容
- 雇用形態（正社員/契約社員/パート）
- 勤務地、勤務時間、給与
- **体験実習の可否**（重要）
- 提供可能な配慮タグの選択

**制限事項（MVP）**

- 無料プラン：1社あたり2求人まで
- 画像アップロード：オフィス写真1枚のみ

**工数**：3日

---

#### 4. マッチング・検索機能（Week 4-5）

**検索機能**

- 求職者が配慮タグでフィルタリング検索
- 職種、勤務地、雇用形態での絞り込み
- 「体験実習可」の求人のみ表示

**マッチ度の計算**

# 簡易マッチングアルゴリズム

def calculate_match_score(job_seeker, job)
required_tags = job_seeker.required_accommodation_tags
provided_tags = job.company.accommodation_tags

matched = required_tags & provided_tags
match_rate = (matched.size.to_f / required_tags.size \* 100).round

return match_rate
end

**表示**

- マッチ度が高い順にソート
- 「マッチ度: 90%」のように表示

**工数**：5日

---

#### 5. 応募・メッセージ機能（Week 5-6）

**求職者側**

- 気になる求人に「応募」ボタン
- 簡易メッセージ送信（志望動機、質問など）

**企業側**

- 応募者一覧の表示
- 応募者のプロフィールと必要な配慮タグを確認
- メッセージでやり取り

**実装**

- Action Cable（リアルタイムチャット）は見送り
- シンプルなスレッド型メッセージ（非リアルタイム）

**工数**：4日

---

#### 6. 体験実習マッチング機能（Week 6-8）

**実習申し込み**

- 企業が応募者に「実習オファー」を送信
- 実習期間（1～2週間）、開始日、勤務時間を提案

**実習中の機能**

- 企業・求職者が日報を記入（任意）
- 支援員もコメント可能（Phase 2で本格化）

**実習後の振り返り**

- 3者で実習を評価（5段階評価 + 自由記述）
- 企業が本採用オファーを送信可能

**実装詳細**
class Internship < ApplicationRecord
belongs_to :job_seeker
belongs_to :company
belongs_to :job

enum status: {
pending: 0, # オファー送信済み
accepted: 1, # 求職者が承諾
in_progress: 2, # 実習中
completed: 3, # 実習完了
hired: 4 # 本採用決定
}
end

**工数**：7日

---

#### 7. ダッシュボード（Week 8-9）

**求職者用ダッシュボード**

- 応募中の求人一覧
- 実習予定・進行中の実習
- マッチ度の高いおすすめ求人

**企業用ダッシュボード**

- 応募者一覧
- 実習中の求職者
- 求人の閲覧数、応募数

**工数**：4日

---

#### 8. 企業向けガイドページ（Week 9-10）

**コンテンツ**

- 障害者雇用の基礎知識
- 配慮の具体例（業種別）
- 助成金情報（厚生労働省の特定求職者雇用開発助成金など）
- よくある質問（FAQ）

**実装**

- 静的ページ（Markdown）
- 外部リンク集

**工数**：2日

---

#### 9. テスト・デバッグ（Week 10-12）

**テスト項目**

- ユーザー登録～マッチング～応募～実習の一連フロー
- 配慮タグのマッチング精度確認
- セキュリティテスト（個人情報保護）

**工数**：6日

---

### Phase 1で意図的に実装しない機能

1. ❌ **支援員の本格機能**（Phase 2へ）
   - 理由：まずは求職者↔企業の直接マッチングを検証
2. ❌ **リアルタイムチャット**（Phase 2へ）
   - 理由：シンプルなメッセージ機能で十分
3. ❌ **定着支援機能**（Phase 3へ）
   - 理由：まずは採用までのフローを確立
4. ❌ **決済機能**（Phase 2へ）
   - 理由：MVPは無料で提供し、ニーズを検証

## 技術スタック

### バックエンド

- **Ruby on Rails 7.1**
  - 理由：MVPの迅速な開発、豊富なgem、Hotwireでモダンなインタラクション
- **PostgreSQL**
  - 理由：リレーショナルデータベースとして安定、Herokuとの親和性

### フロントエンド

- **Hotwire（Turbo + Stimulus）**
  - 理由：Reactなし、Railsのみでモダンなインタラクション実現
- **Tailwind CSS**
  - 理由：迅速なUI構築、デザインシステムとの親和性

### 認証

- **Devise**
  - 理由：Rails標準、複数ユーザータイプの管理が容易

### ホスティング

- **Heroku（または Fly.io）**
  - 理由：デプロイが簡単、初期無料枠あり、スケール容易

### その他

- **Action Mailer**：メール送信（パスワードリセット、通知）
- **ActiveStorage**：画像アップロード（オフィス写真など）

## 収益モデル

### Phase 1（MVP）：無料提供

- 目的：ユーザー獲得、課題検証
- 企業・求職者ともに無料

### Phase 2以降：企業課金モデル

#### 1. 成功報酬型（メイン）

- **採用決定時：30万円**（中途採用の相場の1/3程度）
- **実習経由の本採用：25万円**（割引価格で定着率向上を評価）

#### 2. 月額プラン

- **ライトプラン（月額2万円）**
  - 求人掲載2件
  - 応募者管理
  - 配慮タグマッチング
- **スタンダードプラン（月額5万円）**
  - 求人掲載5件
  - 実習受け入れ無制限
  - 定着支援サポート（Phase 3）

#### 3. 自治体・支援事業所連携

- 自治体の「障害者雇用促進事業」への提案
- 支援事業所からの紹介手数料（1件あたり5～10万円）

### 収益目標（1年後）

- 登録企業：100社
- 月間マッチング成立：10件
- 月間売上：300万円（成功報酬 × 10件）

## リスクと対策

### リスク1：求職者が集まらない

**原因**：認知度不足、既存サービスとの競合  
**対策**：

- 就労移行支援事業所への直接営業（10社）
- SNS広告（精神障害者向けコミュニティ）
- 紹介キャンペーン（登録で Amazonギフト券500円）

### リスク2：企業が求人を掲載しない

**原因**：無料でも手間がかかる、信頼性が不明  
**対策**：

- 初期は営業代行（求人内容を代わりに入力）
- 「障害者雇用はじめてガイド」で教育
- 既存の人事担当者ネットワーク活用

### リスク3：マッチングしても採用に至らない

**原因**：配慮タグだけでは判断できない、面接で不採用  
**対策**：

- 体験実習を必須化（いきなり採用を避ける）
- 支援員の介入強化（Phase 2）
- 企業向けの面接ノウハウ提供

### リスク4：個人情報保護の問題

**原因**：障害情報は機微情報、漏洩リスク  
**対策**：

- プライバシーポリシーの整備
- SSL/TLS暗号化
- 障害者手帳の画像アップロードは不要（自己申告のみ）

## まとめ

### このプロジェクトの強み

1. ✅ **法改正の追い風**：2026年7月の法定雇用率引き上げで需要急増
2. ✅ **明確な差別化**：配慮の可視化 × 体験実習マッチング

4. ✅ **収益性**：成功報酬型で初期コスト不要、スケール可能
